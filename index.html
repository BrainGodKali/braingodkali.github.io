我成功的用Publii创建Github博客

创建 GitHub 页面网站 Publii包括对Github Pages的支持，它为您提供了一个空间来构建一个网站，您可以使用该网站来突出显示Github项目，或者只是您自己！不幸的是，除非您有一些专业知识并愿意学习难以设置的工具（如Jekyll），否则实际构建网站的功能很少，但是多亏了Publii，您将能够在短短几分钟内创建一个功能齐全，令人惊叹的网站，该网站非常易于更新和维护。 创建 Github 页面不会花费很长时间，但我们需要介绍几个步骤，首先是： 在开始创建Github页面之前，我们需要生成一个访问令牌;这将允许Publii访问并更改您的Github页面，以便每当您单击Publii应用程序中的同步按钮时，它都可以创建和编辑您的网站;可以把它想象成创建一个密码，Publii可以用来进入Github俱乐部！ 生成访问令牌只需几秒钟;只需按照以下步骤操作： 请记住：您将无法再次看到令牌，因此请确保将其保存在某个位置。 我们现在生成了一个令牌，因此我们可以在Publii中创建我们的网站，并授予它访问我们的Github页面的权限。 我们已经有了访问令牌，因此这将允许Publii访问我们的Github页面以上传网站文件，但现在我们必须在Github上实际创建我们网站将要进入的区域。让我们回到Github并开始吧： 这就是Github方面的事情;现在我们可以跳到Publii进行最后一步。 在 Github 上准备好所有内容后，我们可以更新 Publii 中的连接设置，以便与我们刚刚创建的 Github Page 存储库同步。首先，开放Publii;如果您尚未创建网站，请创建一个网站并添加至少一个帖子，并确保已选择主题;没有它们，您的网站将无法正确创建。 现在，单击左侧边栏中的“服务器”选项以打开“服务器设置”屏幕。使用以下信息填写字段： 设置完所有选项后，单击“保存设置”按钮以存储更改。现在可能是单击“测试连接”按钮的好时机，该按钮将检查Publii是否能够连接到您的Github页面;如果不是，则某些字段可能输入不正确。 现在一切准备就绪，因此我们可以单击Publii侧边栏中的“同步您的网站”按钮开始部署。 重要！一旦同步过程完成，您上传的Publii网站可能不会立即可见;可能需要大约 1 分钟才能在网站上看到这些更改。如果要仔细检查更改是否已应用，可以检查存储库的“提交”部分，并选中提交区域中最新提交旁边的图标： 如果图标是橙色圆点，则表示已收到更改，但尚未部署。经过短暂的等待，您的更改将被添加。
