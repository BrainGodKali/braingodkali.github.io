我成功的用Publii创建Github博客

BrainGod_Kali

创建 GitHub 页面网站

Publii包括对Github Pages的支持，它为您提供了一个空间来构建一个网站，您可以使用该网站来突出显示Github项目，或者只是您自己！不幸的是，除非您有一些专业知识并愿意学习难以设置的工具（如Jekyll），否则实际构建网站的功能很少，但是多亏了Publii，您将能够在短短几分钟内创建一个功能齐全，令人惊叹的网站，该网站非常易于更新和维护。

创建 Github 页面不会花费很长时间，但我们需要介绍几个步骤，首先是：

生成访问令牌

在开始创建Github页面之前，我们需要生成一个访问令牌;这将允许Publii访问并更改您的Github页面，以便每当您单击Publii应用程序中的同步按钮时，它都可以创建和编辑您的网站;可以把它想象成创建一个密码，Publii可以用来进入Github俱乐部！

生成访问令牌只需几秒钟;只需按照以下步骤操作：

  1. 首先转到Github首页并登录到您的帐户;如果您还没有帐户，只需单击“注册”链接并完成注册过程即可。
  2. 登录后，打开 Github 配置文件设置。
  3. 在个人资料屏幕的左侧栏中，单击“开发人员设置”。
  4. 在 GitHub 应用页面的左侧边栏中，单击“个人访问令牌”选项。
  5. 现在，我们位于“个人访问令牌”页面上;我们需要为我们的网站制作一个新的，所以点击屏幕顶部附近的“生成新令牌”按钮。
  6. 在下一个屏幕上，填写令牌描述部分中的名称，例如“Publii”。现在，在“选择作用域”部分中，向主 Repo 选项添加一个复选标记，这将使令牌能够访问管理存储库的所有方面。最后，单击屏幕底部的绿色“生成令牌”按钮。
  7. 现在，你将返回到“个人访问令牌”页，但现在你的新令牌将显示在列表顶部。无论你做什么，都不要关闭或刷新此页面;令牌仅在生成后立即可见。单击令牌旁边的“复制”图标（请参阅下图），并将其复制到安全的地方，因为我们在设置Publii以使用Github页面时需要它。

    请记住：您将无法再次看到令牌，因此请确保将其保存在某个位置。

我们现在生成了一个令牌，因此我们可以在Publii中创建我们的网站，并授予它访问我们的Github页面的权限。

在 Github Pages 上创建我们的静态网站

我们已经有了访问令牌，因此这将允许Publii访问我们的Github页面以上传网站文件，但现在我们必须在Github上实际创建我们网站将要进入的区域。让我们回到Github并开始吧：

  1. 像往常一样登录Github，然后点击顶栏中的加号图标打开一个子菜单;现在，单击“新建存储库”选项，转到“创建新存储库”屏幕。
  2. 在此屏幕上，填写存储库名称，并将存储库保留为“公共”（Github 页面始终是公共的，即使托管它的存储库是私有的）。接下来，单击屏幕底部的绿色“创建存储库”按钮。
  3. 现在我们进入快速设置屏幕;这可能看起来有点令人困惑，但我们只需要在这里做一件简单的事情。点击自述文件链接，如下图所示;这将带我们进入创建自述文件的屏幕。
  4. 在自述文件的编辑屏幕上，只需单击“提交新文件”按钮。我们之所以创建此文件，是因为您的存储库至少需要添加一个项目来创建主分支，所有 Publii 文件都将位于主分支的位置。此创建的文件将在Publii首次与我们的Github页面同步时自动删除。
  5. 提交文件后，我们将返回到文件列表;单击屏幕顶部附近的“设置”选项卡。
  6. 在设置屏幕上，滚动到Github页面部分;在“源”选项中，单击默认显示“无”的下拉列表，然后从选项列表中选择“主要分支”。现在，单击“保存”按钮以确保已提交更改。
  7. 保存后，您的GitHub页面将准备好用作静态网站主机;要查看您的网站，您需要转到绿色消息框中提供的URL。当然，您的网站还没有任何文件，但我们将对此进行更改。

这就是Github方面的事情;现在我们可以跳到Publii进行最后一步。

将您的Publii静态站点与Github同步

在 Github 上准备好所有内容后，我们可以更新 Publii 中的连接设置，以便与我们刚刚创建的 Github Page 存储库同步。首先，开放Publii;如果您尚未创建网站，请创建一个网站并添加至少一个帖子，并确保已选择主题;没有它们，您的网站将无法正确创建。

现在，单击左侧边栏中的“服务器”选项以打开“服务器设置”屏幕。使用以下信息填写字段：

  • 网站网址 - 这将是您的Github存储库路径，其格式为“YOUR_USERNAME.github.io/ YOUR_REPOSITORY_NAME”。
  • API 服务器 - 默认情况下，此字段已随标准 API 服务器一起完成。但是，如果您正在运行自己的 GitHub 实例，则可以通过在此处输入 API 服务器来更改它。
  • 用户名 - 在此处添加您的 Github 用户名。
  • 存储库 - 在此处输入 Github 存储库的名称（我们在上面的 Github 页面部分的步骤 2 中设置了此名称）。
  • 分支 - 如果您一直遵循我们的指南，请在文本字段中编写主要（区分大小写）;有经验的用户可能希望指定其他设置，例如gh页面，文档等...
  • 令牌 - 还记得我们在本指南第一部分中保存的访问令牌吗？在此处输入该内容，以便 Publii 可以访问存储库。
  • 并行上传 - 默认为“1”。设置 Publii 在将网站内容上传到 GitHub 页面时可以使用的上传流数。数字越大，上传速度越快，但可能会导致连接速度慢时出现错误，或者由于 API 上传速率限制而导致错误 403。
  •  API 速率限制 - 仅当使用禁用 API 速率限制的 Github Enterprise 时，才禁用此选项。其他禁用此选项可能会导致部署错误。

设置完所有选项后，单击“保存设置”按钮以存储更改。现在可能是单击“测试连接”按钮的好时机，该按钮将检查Publii是否能够连接到您的Github页面;如果不是，则某些字段可能输入不正确。

现在一切准备就绪，因此我们可以单击Publii侧边栏中的“同步您的网站”按钮开始部署。

重要！一旦同步过程完成，您上传的Publii网站可能不会立即可见;可能需要大约 1 分钟才能在网站上看到这些更改。如果要仔细检查更改是否已应用，可以检查存储库的“提交”部分，并选中提交区域中最新提交旁边的图标：

如果图标是橙色圆点，则表示已收到更改，但尚未部署。经过短暂的等待，您的更改将被添加。
